# JD Lens — 产品功能清单
> 面向求职场景的 JD 智能分析工具，任何岗位通用
> 版本：v0.3 | 更新时间：2026-05-02

---

## 产品概述

| 项目 | 内容 |
|------|------|
| **产品名称** | JD Lens |
| **目标用户** | 正在求职/跳槽的任意岗位求职者 |
| **核心价值** | 把一份 JD 转化为求职决策所需的全部信息 |
| **输入限制** | JD仅支持文本粘贴；简历支持文本粘贴 / PDF上传 / Word上传 |
| **技术依赖** | 大语言模型（LLM）API |

---

## 一、输入模块

### F-01 JD 原文输入

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **描述** | 用户将招聘JD文本粘贴至输入框 |
| **交互** | 支持粘贴、手动输入、清空 |
| **校验** | 未填写时，右侧生成按钮置灰并提示"请先粘贴JD" |
| **字数限制** | 建议上限 5000 字，超出提示截断风险 |

### F-02 简历输入（三种方式）

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 用户提供简历内容，支持三种输入方式（选填） |
| **交互** | 折叠收起状态，点击展开；Tab切换三种输入方式；支持清空 |
| **说明文字** | "选填：仅「简历匹配」模块需要" |
| **未填时** | 简历匹配 Tab 显示"请先提供简历以启用此功能" |

**三种输入方式（Tab切换）：**

| 方式 | 描述 | 实现复杂度 |
|------|------|----------|
| 📋 **粘贴文本** | 默认方式，手动复制简历文本粘贴 | 🟢 低 |
| 📄 **上传 PDF** | 上传PDF格式简历，前端自动解析为文本 | 🟡 中 |
| 📝 **上传 Word** | 上传.docx格式简历，前端自动解析为文本 | 🟢 低 |

### F-02a 文件解析引擎

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **PDF解析** | 使用 `pdf.js` 在浏览器端完成，无需后端，支持文字版PDF |
| **Word解析** | 使用 `mammoth.js` 在浏览器端完成，无需后端 |
| **不支持格式** | 扫描版PDF（图片型）、JPG/PNG图片格式（需OCR，复杂度高暂不做）|
| **格式说明** | 解析后仅保留纯文本内容，排版格式丢失，但对LLM分析无影响 |
| **解析失败处理** | 解析失败时提示用户改用"粘贴文本"方式 |
| **文件大小限制** | 建议上限 5MB |

### F-03 生成控制

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **按钮一** | `✨ 生成全部` — 同时触发所有 Tab 生成 |
| **按钮二** | `▷ 仅生成当前 Tab` — 只生成当前选中的 Tab |
| **状态** | 生成中显示 Loading 动画；完成后按钮恢复 |

## 二、输出模块（6 个 Tab）

### F-10 岗位解码 🔍

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **用户价值** | 读懂JD背后的真实意图，避免被官方话术误导 |

**输出内容：**

| 子功能 | 描述 |
|--------|------|
| **真实职责还原** | 剥除官方话术，用3-5条白话文描述每天实际要做的事 |
| **岗位性质判断** | 明确区分岗位类型（执行层/管理层/专家型/通才型等） |
| **公司/团队阶段** | 从JD推断团队所处阶段：初创探索 / 高速扩张 / 成熟稳定 |
| **汇报关系推断** | 大概率向谁汇报，在组织中的位置 |
| **隐性要求挖掘** | JD没写但必然会考察的潜台词 |

---

### F-11 典型一天 ☀️

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **用户价值** | 直观感受入职后的真实工作状态，判断是否与自身期望匹配 |

**输出内容（6个固定板块，顺序不可变）：**

| 板块 | 描述 |
|------|------|
| **岗位定位** | 2-3句话描述该角色在团队中的核心价值 |
| **一天时间表** | 精确到小时的工作日模拟，格式：`09:30 → 做什么 → 产出什么` |
| **一周节奏** | 周一至周五的工作侧重点分布 |
| **高频任务清单** | 列举5-8项出现频率最高的工作任务 |
| **主要协作对象** | 需要频繁对接的角色 |
| **工作强度信号** | 从时间表推断：加班风险 / 会议密度 / 独立工作比例 |

---

### F-12 技能拆解 🧠

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **用户价值** | 知道该重点准备什么，面试前针对性补强 |

**输出内容：**

| 子功能 | 描述 |
|--------|------|
| **技能维度权重** | 以文字展示各维度权重（根据岗位类型动态调整维度） |
| **必备技能清单** | 硬性要求，不具备即被筛掉 |
| **加分项识别** | 具备后会让HR/面试官眼前一亮的经验 |
| **隐性门槛** | JD未提及但实际面试会考的能力 |
| **学习建议** | 针对此岗位，建议重点提升的1-2个方向 |

---

### F-13 简历匹配 📄

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **前置条件** | 用户已提供简历（F-02） |
| **用户价值** | 客观评估简历竞争力，知道怎么改才能提升通过率 |

**输出内容：**

| 子功能 | 描述 |
|--------|------|
| **综合匹配评分** | 0-100分，含分项得分（技能/经验/行业/背景） |
| **优势亮点** | 简历中最能打动该岗位面试官的3条内容 |
| **短板分析** | 与JD要求的差距，以及是否属于致命短板 |
| **简历优化建议** | 具体到哪几句话怎么改，突出与JD的关键词匹配 |
| **一句话pitch** | AI生成针对该岗位的个性化自我介绍开场白 |

---

### F-14 面试准备 🎯

| 属性 | 内容 |
|------|------|
| **优先级** | P2 |
| **用户价值** | 省去自己研究面试题的时间，针对性准备 |

**输出内容：**

| 子功能 | 描述 |
|--------|------|
| **高概率面试题** | 根据JD生成5-8道最可能被问到的问题 |
| **答题思路框架** | 每道题给出结构化回答思路（STAR法则/金字塔原则） |
| **岗位专项题** | 针对该类岗位特有的考察点 |
| **反问清单** | 3-5个显示专业度的面试官提问，避免冷场 |

---

### F-15 综合评估 🚦

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **用户价值** | 给出明确的投递建议，帮助快速决策 |

**输出内容：**

| 子功能 | 描述 |
|--------|------|
| **投递价值评级** | ⭐⭐⭐⭐⭐ 强烈推荐 / ⭐⭐⭐ 值得一试 / ⭐ 慎重考虑 |
| **岗位核心亮点** | 这个岗位比同类有哪些优势（成长空间/技术氛围/影响力） |
| **潜在风险预警** | JD中透露的隐患（如：要求过多/KPI压力大/晋升通道不清晰） |
| **薪资参考范围** | 基于岗位级别和要求推测市场薪资区间 |
| **一句话结论** | 直接给出"要不要投"的明确建议及核心理由 |

## 四、Prompt 配置中心 ⚙️

### F-30 Prompt 配置入口

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **入口位置** | 页面右上角 `⚙ Prompt配置` 按钮 |
| **交互形式** | 点击弹出模态框（Modal） |

### F-31 分 Tab Prompt 编辑

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **描述** | 配置面板内按输出模块分 Tab，每个Tab对应一个独立的Prompt文本 |
| **可编辑项** | 岗位解码 / 典型一天 / 技能拆解 / 简历匹配 / 面试准备 / 综合评估 |
| **编辑区** | 大文本框，支持多行编辑，展示当前使用的完整Prompt |

### F-32 保存 & 恢复默认

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |
| **保存按钮** | 将修改后的Prompt存储到本地（localStorage），下次生成自动使用 |
| **恢复默认** | 将当前Tab的Prompt重置为系统内置默认值，需二次确认 |
| **提示文案** | "修改后点保存，下一次生成立即使用最新 Prompt" |

### F-33 Prompt 中的变量支持（P2）

| 变量 | 含义 |
|------|------|
| `{{JD}}` | 用户粘贴的JD原文 |
| `{{简历}}` | 用户提供的简历内容 |

## 六、非功能性要求

| 项目 | 要求 |
|------|------|
| **响应时间** | 单个Tab生成时间 < 15秒 |
| **数据存储** | Prompt配置存本地localStorage，无需登录 |
| **文件上传** | 简历支持PDF / .docx上传（前端解析），JD仅支持文本粘贴 |
| **移动端** | 优先PC端体验，移动端基础可用即可 |
| **语言** | 界面中文，AI输出中文 |
